# MVP 开发路线图（V2：宠物孵化）

## 阶段 1：网页孵化闭环（当前优先，3-5 天）

目标：完成“网页导入聊天记录 -> AI 学习 -> 返回宠物大脑”的闭环。

任务：

- 搭建 Next.js 页面（输入区、上传区、学习按钮、结果面板）
- 搭建 FastAPI 的 `/api/v1/hatch` 接口
- 接入 prompt.md 并得到结构化 brain_state JSON
- 返回并渲染成长阶段与里程碑

验收标准：

- 用户能提交聊天记录
- 后端返回合法 brain_state
- 页面能展示“第一次开口说话”等里程碑条目

## 阶段 2：增量学习（2-4 天）

目标：宠物支持继续喂养并更新大脑。

任务：

- 实现 `/api/v1/learn`
- 设计 brain_state 合并策略（保留稳定特征 + 更新新特征）
- 前端增加“继续喂养”入口

验收标准：

- 同一个 brain_id 可多次学习
- growth_model 有可见进度变化

## 阶段 3：C++ 亮点接入（3-5 天）

目标：让语言统计由 C++ 承担，提高性能与稳定性。

任务：

- 实现 C++ 文本特征提取程序
- Python 通过 subprocess JSON 调用
- 将统计结果注入 prompt 上下文

验收标准：

- catchphrases/punctuation/句长分布来自 C++ 结果
- 大文本输入响应时间明显下降

## 阶段 4：桌面化准备（未来计划）

目标：为桌面宠物迁移准备能力边界。

任务：

- 抽离前端状态与大脑状态管理
- 规划 Electron/Tauri 迁移方案
- 定义宠物事件系统（开口、撒娇、主动提醒）

验收标准：

- Web 版本与桌面版本共享核心后端接口
- 里程碑系统可复用
